## 1話~10話
1. インクリメント：変数の値を1増やす操作
   ![[Pasted image 20260702150839.png]]
2. プログラマーの三大美徳：怠慢、短気、傲慢
3. アセンブリ言語：コンピュータ(CPU)が直接理解できる機械語を人間が読み書きしやすい英単語や記号に置き換えたプログラミング言語
4. 機械語：コンピュータの頭脳であるCPUが直接理解して実行できる、最も基本的なプログラミング言語
5. エディタ：一般的にコンピュータ上でテキストや画像、動画などのデータを編集、作成、加工するソフトウェアの総称。
   ex) Vim, Emacs etc 
6. ニ進数：0と1の２種類の数字のみですべての数を表現する記数法
7. しゃくとり法：配列や文字列などの中から「条件を満たす連続した部分列」を効率的に探し出すための探索アルゴリズム。
8. 累積和：配列の「先頭からの合計」をあらかじめ計算し、メモしておくデータ処理のテクニック
## 11話～20話
1. コードゴルフ：プログラミングコンテストの一種。与えられた処理やアルゴリズムを最も短いソースコードで実装することを競う遊び。
2. ペアプログラミング：2人のプログラマーが1台のPCを共有し、協力してプログラムを開発する手法
3. ドライバ：マウスやプリンターなどの周辺機器やPCパーツを、OSが正しく認識・制御するための「翻訳ソフト(専用プログラム)」のこと
4. 巡回セールスマン問題(TSP)：複数の都市をそれぞれ一度だけ訪れて出発地に戻る経路のうち、移動距離やコストが最小になるルートを求める問題
5. ダイクストラ法：ある地点から他のすべての地点(ノード)までの「最短経路」を計算するためのアルゴリズム。カーナビや乗換案内などに活用されている
6. Rubyで素因数分解：標準のPrimeライブラリにあるprime_divisionメソッドを使うことで誰でも超簡単に素因す分解ができる
7. Lisp：リストを主に扱っている高級プログラミング言語。データやコードをすべて括弧で囲った式として表す関数型プログラミング言語。大量の括弧が使わないといけない。
## 21話~30話
1. オープンソース：ソフトウェアの設計図である「ソースコード」が無償で一般公開され、誰でも自由に利用、改良、再配布できる仕組みや開発モデル
2. PHP：主にWeb開発で使用されるオープンソースのサーバーサイド・スクリプト言語。HTMLのに直接埋め込むことができ、動的なWebページ作成やデータベースとの連携に優れている。
3. AI：人間の脳が行っている学習、認識、予測や計画のような知的作業をコンピュータで実現できるようしたソフトウェアやシステムのこと
4. C言語：1972年に開発された非常に歴史の長い汎用プログラミング言語。処理速度が圧倒的に早く、コンピュータのハードウェアを直接制御できる。
# 31話～40話
1. アルゴリズム：特定の問題を解決するための手順や計算方法のこと。
2. Unity：ゲーム開発・３Dコンテンツ制作のための統合開発プラットフォーム(ゲームエンジン)
3. 閾値(しきい値)：ある物事が「反応する」「変化する」「アラートを出す」ための境界となる基準値
4. セグメンテーションフォールト：プログラムがアクセスを許可されていないメモリ領域に触れようとした際、OSによってい強制終了されるエラー。C言語とかでよく見られる現象。
5. フォントムインサート：データベースのトランザクション処理において、あるトランザクションが条件に一致するデータを読み込んでいる最中に、別のトランザクションが同じ条件に該当するデータを新たに割り込ませてしまう現象
6. ガウス＝ルジャンドルのアルゴリズム：円周率を極めて高速に計算できる反復計算アルゴリズム
7. リファクタリング：プログラムの外部から見た動作を変えずに、ソースコードの内部構造を整理し、改善する作業
# 41話～50話
1. 無限ループ：コンピュータプログラムにおいて、繰り返し処理の終了条件が満たされず、処理が延々と反復サれ続けて停止しなくなる現象
2. コンソール：コンピュータやシステムを操作・監視・設定するための入出力装置やそのための画面のこと
# 51話～60話
1. フローチャート：プロセスや業務、コンピュータ・プログラムの手順を図形や矢印を用いて視覚的に表現した図
2. クラウド：手元の端末にソフトウェアやデータを保存するのではなくインターネットを通じて、提供されるサーバーやアプリを必要なときに必要なだけ利用する仕組み
3. スパゲッティコード：コンピュータ・プログラムの構造や処理の流れが複雑に入り組み、まるで茹で上がったスパゲッティのように絡み合って解読困難になっている状態を指すプログラミング用語
4. ラバーダッキング：何かに対して問題を話すことで、頭が整理されて解決法が見つかるエンジニアのテクニック
5. ポインタ：プログラミングにおけるメモリの番地(アドレス)を指し示す変数
6. グローバル変数：プログラム全体からアクセスし、値を読み書きできる変数。プログラム全体からいつでも参照・変更ができる
7. ブーリアン型：TrueかFalseの2つの値のみ扱うプログラミングのデータ型
8. ターミナル：キーボードなどの入力装置から文字でコンピュータに命令を送り結果を文字で受け取るためのツール
9. ビット演算子：数値をコンピュータの最小単位である「0」と「1」のビット列として直接処理するための演算子
10. スニペット：よく使う構文を登録しておき短いキーワードを入力するだけで呼び出せるようにする
11. ドメイン特化言語：特定の用途や問題解決に特化して設計されたコンピュータ言語
